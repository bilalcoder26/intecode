What is backpressure in streams?

**Backpressure** is a mechanism that prevents a fast data producer from overwhelming a slower data consumer.

In simple terms:

> Backpressure occurs when data is being generated faster than it can be processed or consumed.

---

## Real-Life Example

Imagine a water pipe:

Water Tank
    |
    | 100 L/min
    v
 Pipe
    |
    | 10 L/min
    v
 Bucket

The tank sends water much faster than the bucket can accept it.

Result:

Overflow ❌

Backpressure is the mechanism that tells the tank:

"Slow down, the bucket isn't keeping up."

# How Node.js Handles It

The `write()` method returns a boolean.

### When Buffer Has Space

writable.write(chunk);

returns:

true

Meaning:

Keep sending data

---

### When Buffer Is Full

const canContinue = writable.write(chunk);

returns:

false

Meaning:

Stop sending data

---

# Manual Backpressure Example

const fs = require('fs');

const readStream = fs.createReadStream('input.txt');
const writeStream = fs.createWriteStream('output.txt');

readStream.on('data', (chunk) => {
    const canWrite = writeStream.write(chunk);

    if (!canWrite) {
        readStream.pause();
    }
});

writeStream.on('drain', () => {
    readStream.resume();
});

# The `drain` Event

When the writable stream has processed enough data and has room again:

writeStream.on('drain', () => {
    console.log('Buffer available');
});

Node.js emits the `drain` event.

---

# Using `pipe()`

Most of the time, you don't handle backpressure manually.

readStream.pipe(writeStream);

Node.js automatically:

* Pauses the readable stream
* Waits for the writable stream
* Resumes when ready

Read Stream
      |
     pipe()
      |
Write Stream

Backpressure handling is built in.

# Interview Answer (Short)

> Backpressure is a flow-control mechanism in Node.js streams that prevents a fast producer from overwhelming 
a slower consumer. When a writable stream's buffer becomes full, `write()` returns `false`, signaling the readable 
stream to pause. Once the buffer is drained, the `drain` event is emitted, allowing the readable stream to resume. 
The `pipe()` method handles backpressure automatically, ensuring efficient memory usage and stable performance 
when processing large amounts of data.
