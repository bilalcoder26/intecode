What is Middleware in Express?

Middleware functions are functions that have access to the request object (req), response object (res), 
and the next function in the application's request-response cycle.

Think of middleware as a processing pipeline - each request passes through a series of functions that can:

Execute code

Modify req or res objects

End the request-response cycle

Call the next middleware function


Basic Middleware Structure

function myMiddleware(req, res, next) {
  // Do something
  console.log('Middleware executed');
  
  // Pass control to next middleware/route handler
  next();
}

app.use(myMiddleware);

The next() Function
With next(): Continues to the next middleware/route

Without next(): Request gets stuck (unless response is sent)

With error: next(err) jumps to error-handling middleware

Key Takeaways
Middleware functions have req, res, and next parameters

Call next() to pass control to the next middleware

Order matters - define middleware before routes that need them

Error middleware has 4 parameters and goes at the end

You can end the cycle by sending a response without calling next()

Middleware is what makes Express powerful and extensible - 
it's the secret sauce that allows you to build complex web applications with clean, modular code.


Middleware in Express is a function that has access to the request object (req), response object (res), 
and the next() function. It executes during the request-response cycle and can modify requests, responses, 
perform authentication, logging, validation, error handling, or terminate the request. 
The main types are Application-level, Router-level, Built-in, Error-handling, and Third-party middleware.
